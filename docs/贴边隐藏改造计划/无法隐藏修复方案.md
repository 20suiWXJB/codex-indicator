问题诊断

你的贴边隐藏功能代码已经全部实现完成,但窗口实际上一直停留在 peek 模式,从未进入 hidden 模式。

核心原因

查看 src/mainPanel.ts:443-455,鼠标离开触发隐藏的逻辑:

document.documentElement.addEventListener("mouseleave", () => {
if (peekTimer !== undefined) {
window.clearTimeout(peekTimer);
peekTimer = undefined;
}
if (dockUi.mode === "peek") {
hideTimer = window.setTimeout(() => {
hideTimer = undefined;
void dispatchDock({ kind: "pointerLeaveSettled" });
}, settings.dockHideDelayMs);
}
});

问题在于:

1. document.documentElement.addEventListener("mouseleave") 监听点错误
- documentElement 是 <html> 元素
- 在 Tauri 的透明窗口中,当鼠标在窗口内部的透明区域移动时,浏览器认为鼠标仍在 <html> 内,不会触发 mouseleave
- 只有当鼠标完全离开窗口边界才会触发,但此时鼠标可能已经移到屏幕边缘外了
2. 贴边场景下的特殊性
- 贴边 peek 模式时,窗口紧贴屏幕边缘
- 用户鼠标移开时,通常是平行于边缘方向滑走(例如右侧贴边时向下移动)
- 鼠标路径可能始终在窗口的垂直投影内,导致 mouseleave 永远不触发
3. 你的灯珠和胶囊布局
   查看 CSS:
   .dock-bead {
   width: 100vw;
   height: 100vh;  /* 填满整个窗口 */
   }
3. 灯珠填满了整个 14×56 的小窗口,理论上鼠标离开小窗口应该触发 mouseleave,但实际测试中这个事件不可靠。

解决方案

方案 1: 监听窗口焦点失去事件(推荐)

不依赖 mouseleave,改用 Tauri 的窗口焦点事件:

修改位置: src/mainPanel.ts:443-455

// 删除原有的 mouseleave 监听,改用窗口 blur 事件
try {
void getCurrentWindow().onFocusChanged(({ payload: focused }) => {
if (!focused) {
// 窗口失去焦点,说明用户鼠标移开了
if (peekTimer !== undefined) {
window.clearTimeout(peekTimer);
peekTimer = undefined;
}
if (dockUi.mode === "peek") {
hideTimer = window.setTimeout(() => {
hideTimer = undefined;
void dispatchDock({ kind: "pointerLeaveSettled" });
}, settings.dockHideDelayMs);
}
} else {
// 窗口获得焦点,取消隐藏计时器
if (hideTimer !== undefined) {
window.clearTimeout(hideTimer);
hideTimer = undefined;
}
}
}).catch((error) => {
console.warn("window focus listener registration failed:", error);
});
} catch (error) {
console.warn("window focus listener registration failed:", error);
}

原理:
- Tauri 窗口默认 focusable: true
- 鼠标悬停在窗口上时窗口自动获得焦点
- 鼠标移开后,窗口失去焦点,触发 focused: false
- 比 DOM 的 mouseleave 更可靠

方案 2: 混合监听(更保险)

同时监听 mouseleave 和窗口焦点,取两者的或逻辑:

// 状态标记:是否应该隐藏
let shouldHideWhenSettled = false;

// 监听鼠标离开
document.documentElement.addEventListener("mouseleave", () => {
if (peekTimer !== undefined) {
window.clearTimeout(peekTimer);
peekTimer = undefined;
}
shouldHideWhenSettled = true;
scheduleDockHide();
});

// 监听窗口失焦
try {
void getCurrentWindow().onFocusChanged(({ payload: focused }) => {
if (!focused) {
if (peekTimer !== undefined) {
window.clearTimeout(peekTimer);
peekTimer = undefined;
}
shouldHideWhenSettled = true;
scheduleDockHide();
} else {
shouldHideWhenSettled = false;
if (hideTimer !== undefined) {
window.clearTimeout(hideTimer);
hideTimer = undefined;
}
}
}).catch((error) => {
console.warn("window focus listener registration failed:", error);
});
} catch (error) {
console.warn("window focus listener registration failed:", error);
}

function scheduleDockHide() {
if (hideTimer !== undefined) {
return; // 已经在倒计时
}
if (dockUi.mode === "peek" && shouldHideWhenSettled) {
hideTimer = window.setTimeout(() => {
hideTimer = undefined;
if (shouldHideWhenSettled) { // 二次确认
void dispatchDock({ kind: "pointerLeaveSettled" });
shouldHideWhenSettled = false;
}
}, settings.dockHideDelayMs);
}
}

方案 3: 主动轮询鼠标位置(最可靠但稍重)

如果上述两种方案仍有问题,可以在 peek 模式下启动一个轻量级轮询:

let mousePosCheckTimer: number | undefined;

// mouseenter 时启动检测
document.documentElement.addEventListener("mouseenter", () => {
// ... 原有 peek 逻辑
startMousePosCheck();
});

// mouseleave 时作为后备
document.documentElement.addEventListener("mouseleave", () => {
stopMousePosCheck();
// ... 原有 hide 逻辑
});

function startMousePosCheck() {
stopMousePosCheck();
if (dockUi.mode !== "peek") return;

let lastX = 0, lastY = 0;
document.documentElement.addEventListener("mousemove", (e) => {
lastX = e.clientX;
lastY = e.clientY;
});

mousePosCheckTimer = window.setInterval(async () => {
if (dockUi.mode !== "peek") {
stopMousePosCheck();
return;
}

    // 检查鼠标是否在窗口内
    const bounds = document.documentElement.getBoundingClientRect();
    const isInside = lastX >= 0 && lastX <= bounds.width &&
                     lastY >= 0 && lastY <= bounds.height;

    if (!isInside) {
      stopMousePosCheck();
      // 触发隐藏
      if (hideTimer === undefined) {
        hideTimer = window.setTimeout(() => {
          hideTimer = undefined;
          void dispatchDock({ kind: "pointerLeaveSettled" });
        }, settings.dockHideDelayMs);
      }
    }
}, 100); // 100ms 检测一次
}

function stopMousePosCheck() {
if (mousePosCheckTimer !== undefined) {
window.clearInterval(mousePosCheckTimer);
mousePosCheckTimer = undefined;
}
}

我的推荐

先尝试方案 1(窗口焦点监听),因为:
1. 代码量最小
2. 性能最好(事件驱动,不占用 CPU)
3. 对 Tauri 透明窗口最可靠

如果方案 1 在你的系统上仍有问题,再升级到方案 2(混合监听)。

方案 3 是最后的保底方案,只在前两者都失效时使用。
