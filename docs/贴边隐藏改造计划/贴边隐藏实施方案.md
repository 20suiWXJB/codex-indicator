# 贴边隐藏（Edge Dock）实施方案

> 目标：主悬浮窗拖到屏幕边缘后自动"收纳"，只留一颗贴边小灯珠，灯珠颜色/动效反映 Codex 聚合状态；鼠标悬停滑出完整胶囊，拖离边缘恢复普通悬浮。交互范式对齐 QQ 贴边收纳。

---

## 1. 需求本质

当前指示器永远占据一块 220×72 的屏幕区域，但它 95% 的时间只在传递一个比特的信息："Codex 现在什么状态"。这个信息用一个颜色点就能表达——文字、摘要、按钮只在用户主动关注时才有价值。

所以本质需求是：**把"常驻信息密度"降到一颗灯珠，把"完整信息"变成按需滑出**。

- 常驻态（贴边灯珠）：只传递聚合状态色 + 动效（运行呼吸 / 等待脉冲 / 异常闪烁），其他信息全部省略。
- 按需态（悬停滑出）：现有胶囊原样滑出，所有功能（展开事件、设置、置顶、隐藏）不变。
- 自由态（拖离边缘）：和现在完全一样的悬浮窗。

## 2. 交互规范（定稿）

### 2.1 三个模式，一个窗口

```
                拖拽结束且贴近边缘(≤12px)
   ┌──────────┐ ───────────────────────────▶ ┌──────────────┐
   │  normal  │                              │ docked-peek  │ (胶囊贴边完整可见)
   │ 普通悬浮  │ ◀─────────────────────────── │              │
   └──────────┘    拖拽结束且远离边缘          └──────────────┘
                                               │          ▲
                             鼠标离开窗口且     │          │ 鼠标移入灯珠
                             超过收起延迟(600ms)│          │ 且停留 120ms
                                               ▼          │
                                             ┌──────────────┐
                                             │docked-hidden │ (只剩灯珠)
                                             └──────────────┘
```

状态转移规则：

| 当前模式 | 触发 | 目标模式 | 说明 |
| --- | --- | --- | --- |
| normal | 拖拽结束，窗口边缘距工作区左/右/上边缘 ≤ 12px | docked-peek | 窗口吸附贴平边缘；松手后鼠标还在窗口上，所以先进 peek，靠离开计时器自然收起 |
| docked-peek | 鼠标离开窗口持续 ≥ 收起延迟（默认 600ms） | docked-hidden | 计时期间鼠标移回则取消；收起时强制关闭事件面板 |
| docked-hidden | 鼠标移入灯珠停留 ≥ 120ms | docked-peek | 120ms 防误触（鼠标快速划过屏幕边缘不该弹出） |
| docked-hidden | 按住灯珠拖拽 | （拖拽中） | pointerdown 取消 peek 计时器，直接走 `startDragging()` |
| docked-peek / 拖拽中 | 拖拽结束，远离边缘（> 12px） | normal | 恢复 220×72，位置就是松手位置 |
| 任意 docked | 设置里关闭"贴边收纳" | normal | 后端把窗口从边缘内推到完整可见位置 |

### 2.2 几何规格（逻辑像素）

| 项 | 值 | 说明 |
| --- | --- | --- |
| 吸附判定阈值 `SNAP_THRESHOLD` | 12 | 窗口外沿到工作区边缘的距离 |
| 拖拽结束判定 `MOVE_DEBOUNCE_MS` | 400 | 最后一次 `onMoved` 事件后 400ms 无新事件视为拖拽结束 |
| 灯珠窗口（左/右边缘） | 14 × 56 | 竖向小条，纵坐标保持收纳前位置（clamp 进工作区） |
| 灯珠窗口（上边缘） | 56 × 14 | 横向小条，横坐标保持收纳前位置 |
| peek 窗口 | 220 × 72（面板展开时 220 × panelExpandedHeight） | 与现有尺寸一致，贴平所在边缘 |
| 悬停展开延迟 `PEEK_DELAY_MS` | 120 | 固定值 |
| 离开收起延迟 `dockHideDelayMs` | 600（可配置 200–3000） | 新设置项 |

支持的边缘：**左、右、上**。不支持下边缘——Windows 任务栏默认在下，贴下边缘的灯珠会紧贴任务栏且与其视觉混淆，QQ 同样只支持左/右/上。

角落场景（同时贴近两条边）：取距离更近的边；等距时优先左/右（竖向灯珠在屏幕侧边更符合直觉，也不与浏览器标签页等顶部 UI 抢位置）。

### 2.3 灯珠视觉规格

灯珠 = 半嵌入边缘的圆头胶囊，视觉上像一颗嵌在屏幕边框上的 LED：

- 底座：14×56 深色圆角条（右侧收纳时只圆左侧两角，`border-radius: 7px 0 0 7px`），沿用胶囊底色 `rgba(20,24,29,0.82)` + 1px 描边。
- 灯芯：居中 6×28 圆头竖条（上边缘收纳时为 28×6 横条），颜色 = 聚合状态色，带 `box-shadow` 光晕。
- 动效与主灯完全一致的语义：
  - `running`：呼吸（复用 `lamp-breath` 的节奏参数 `--breath-period`）
  - `waiting`：1.2s 脉冲（醒目，等待是最需要用户注意的状态）
  - `error`：复用 0.8s `error-pulse` 快闪
  - `interrupted`：静态橙色
  - `done`：绿色，`showDoneSettleMs` 后随聚合状态回落灰色
  - `idle`：40% 透明度灰色（存在感最低）
  - `connecting`：复用 `connecting-pulse`
- 状态切换闪烁（`statusBlinkEnabled`）同样作用于灯芯。
- `prefers-reduced-motion: reduce` 下全部动效降级为静态色。
- 灯珠 `title` 提示当前状态文案（悬停系统 tooltip 兜底，纯附赠）。

## 3. 关键设计决策与理由

每条决策都回答"为什么"：

1. **单窗口三模式，不新建第二个灯珠窗口。**
   状态渲染、设置监听、轮询循环已经全在 main 窗口里；第二个窗口意味着两份轮询/两份状态同步/两个 webview 的内存。收纳只是同一个窗口的"几何 + 渲染形态"变化，改 CSS 数据属性 + 窗口尺寸即可。

2. **几何计算与执行放 Rust，交互状态机放前端。**
   跟现有分工一致（`set_panel_open` 就是前端发意图、Rust 改几何）。Rust 能拿到 `current_monitor()` 的物理尺寸、缩放因子和工作区，多显示器/DPI 计算在后端做一次即可；而悬停、计时器、防误触这些交互细节天然属于前端。核心几何函数写成纯函数，可单测。

3. **拖拽结束检测用 `onMoved` 去抖 400ms，不用全局鼠标钩子。**
   `startDragging()` 进入系统拖拽模态循环期间 webview 收不到 pointer 事件，无法直接感知"松手"。去抖是零依赖、跨平台的近似方案；400ms 内不动就判定结束。误判场景（用户按住不动 400ms 后继续拖）后果只是窗口提前吸附，再拖走即可恢复，代价可接受。P2 可选增强：Windows 下用 `GetAsyncKeyState(VK_LBUTTON)` 确认左键已松开再吸附（需加 `windows-sys` 依赖，MVP 不做）。

4. **去掉 `tauri.conf.json` 里 main 窗口的 min/max 尺寸约束，尺寸全部代码控制。**
   灯珠 14×56 远小于现有 `minWidth: 220 / minHeight: 72`，不删约束 `set_size` 会被系统钳制。窗口本来就 `resizable: false`，用户无法手动改尺寸，min/max 约束没有存在价值——三种模式的尺寸各自由 `set_panel_open` / `set_dock_mode` 显式设置。

5. **收纳状态持久化到独立的 `config/dock-state.json`，不进 `settings.json`。**
   "现在收纳在右边缘 y=300" 是运行时 UI 状态，不是用户偏好；混进 AppSettings 会被设置页的保存流程整体回写，且要陪跑 normalize/契约测试。独立小文件由 Rust 在 dock/undock 时写入，启动时读取恢复。进 settings.json 的只有两个真正的偏好：`edgeDockEnabled` 和 `dockHideDelayMs`。

6. **`tauri-plugin-window-state` 限制为只记录位置（`StateFlags::POSITION`）。**
   该插件默认还会恢复尺寸——重启后可能把窗口恢复成 14×56 的灯珠尺寸但模式是 normal，出现"永远只剩一条缝"的破碎状态。位置仍值得记（对应现有"记住窗口位置"设置），尺寸由启动流程按 dock-state 显式归位。

7. **状态色抽成 CSS 变量，灯珠零逻辑复用。**
   现在状态色硬编码在 `.lamp[data-status=...]` 各条规则里（`style.css:296-327`）。抽成 `.shell[data-status=...] { --status-color: ... }` 后，主灯、灯珠灯芯共用一份色板；未来新增状态只改一处。`shell.dataset.status` 在 `mainPanel.ts:132` 已经每轮更新，灯珠渲染不需要任何新 JS 逻辑。

8. **悬停展开而不是点击展开。**
   对齐 QQ 心智；且灯珠只有 14px 宽，点击目标太小，悬停是更低成本的手势。120ms 停留判定挡住"鼠标甩到屏幕边"的误触。

## 4. 分阶段实施

### 阶段 1：Rust 后端 —— 几何核心 + 窗口命令 + 持久化

全部改动在 `src-tauri/` 内。

#### 1.1 `tauri.conf.json`

删除 main 窗口的 `minWidth` / `minHeight` / `maxWidth` / `maxHeight` 四行（现 `tauri.conf.json:18-22`）。其余不动。

#### 1.2 新模块 `src-tauri/src/dock.rs`（几何纯函数，可单测）

```rust
/// 收纳边缘
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "lowercase")]
pub enum DockEdge { Left, Right, Top }

/// 收纳模式：Hidden=只剩灯珠, Peek=胶囊贴边完整可见
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "lowercase")]
pub enum DockMode { Hidden, Peek }

/// 逻辑像素矩形，(x, y) 为左上角
#[derive(Debug, Clone, Copy, PartialEq)]
pub struct Rect { pub x: f64, pub y: f64, pub w: f64, pub h: f64 }

pub const SNAP_THRESHOLD: f64 = 12.0;
pub const BEAD_THICKNESS: f64 = 14.0;   // 灯珠贴边方向厚度
pub const BEAD_LENGTH: f64 = 56.0;      // 灯珠沿边方向长度
pub const PILL_W: f64 = 220.0;
pub const PILL_H: f64 = 72.0;

/// 判定窗口是否贴近某条可收纳边缘。
/// 同时贴近两条边时取距离更近者，等距优先左/右。
pub fn detect_dock_edge(win: Rect, work: Rect) -> Option<DockEdge> { ... }

/// 计算目标几何。cross 是沿边方向的期望坐标（左/右边缘=y，上边缘=x），
/// 会被 clamp 进工作区，保证灯珠/胶囊完整可见。
pub fn dock_geometry(edge: DockEdge, mode: DockMode, cross: f64, work: Rect, pill_h: f64) -> Rect { ... }
```

要点：

- `dock_geometry` 的 `pill_h` 参数支持 peek 时事件面板展开（传 `panelExpandedHeight`），Hidden 模式忽略。
- 全部用**逻辑像素**计算；命令层用 `monitor.work_area()`（tauri 2 的 `Monitor` 自带工作区，已排除任务栏）除以 `scale_factor()` 转逻辑值，应用时用 `LogicalPosition` / `LogicalSize`，与现有 `set_panel_open`（`lib.rs:1440`）保持同一套单位。
- `work_area` 取窗口当前所在显示器（`window.current_monitor()`），拿不到时回退主显示器。

#### 1.3 收纳状态持久化

```rust
/// config/dock-state.json —— 运行时收纳状态（非用户偏好，不进 settings.json）
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct DockState {
    pub edge: DockEdge,
    /// 沿边方向坐标（逻辑像素），恢复时重新 clamp
    pub cross: f64,
    /// 所在显示器名，找不到时回退主显示器
    pub monitor: Option<String>,
}
```

- `dock_state_path()` → `app_data_root().join("config").join("dock-state.json")`。
- 进入 docked 时写入；undock 时删除文件。读失败/字段非法 → 视为未收纳（不弹错，行为不劣于现状）。

#### 1.4 新增 Tauri 命令（`lib.rs`）

```rust
/// 拖拽结束后由前端调用：检测是否应吸附。
/// 返回 Some(edge) 表示已吸附并完成贴边（peek 几何），None 表示应处于 normal。
#[tauri::command]
fn dock_check(app: AppHandle, panel_expanded_height: Option<f64>) -> Result<Option<DockEdge>, String>

/// 已收纳状态下切换 hidden/peek 几何（悬停展开、离开收起时由前端调用）
#[tauri::command]
fn set_dock_mode(app: AppHandle, mode: DockMode, panel_expanded_height: Option<f64>) -> Result<(), String>

/// 解除收纳：把窗口从边缘内推到完整可见（用于设置关闭贴边、异常恢复），恢复 220×72
#[tauri::command]
fn undock_window(app: AppHandle) -> Result<(), String>

/// 启动/显示时查询持久化的收纳状态，前端据此恢复渲染形态
#[tauri::command]
fn get_dock_state() -> Option<DockState>
```

实现要点：

- `dock_check`：读窗口 `outer_position` + `outer_size` + 所在显示器 → 转逻辑 Rect → `detect_dock_edge`。命中：吸附为 **peek** 几何（松手时鼠标必在窗口上，由前端的离开计时器负责收起）、写 dock-state.json；未命中：若之前是 docked 则删 dock-state 并恢复 220×72 尺寸（位置保持松手处）。设置 `edgeDockEnabled=false` 时直接返回 `Ok(None)` 不做任何吸附。
- `set_dock_mode(Hidden)`：应用灯珠几何。`set_dock_mode(Peek)`：应用胶囊几何。两者都从 dock-state.json 读 edge/cross，文件缺失时返回 Err（前端收到后回退 normal 渲染）。
- 幂等：目标几何与当前一致时直接返回，避免 `onMoved` 反馈循环。

#### 1.5 与现有逻辑的三处对接

1. **`set_panel_open`（`lib.rs:1440`）**：docked-peek 下展开事件面板时窗口要贴边加高。改为：若存在 dock-state 且当前为 peek，几何走 `dock_geometry(edge, Peek, cross, work, expanded_height)`；否则维持现有逻辑。
2. **启动流程（`run()` 的 `.setup`，`lib.rs:1581`）**：
   - window-state 插件改为 `tauri_plugin_window_state::Builder::new().with_state_flags(StateFlags::POSITION)`（理由见决策 6）。
   - setup 里读 dock-state.json：存在 → 应用 Hidden 几何（前端首帧再通过 `get_dock_state` 对齐渲染形态）；不存在 → 显式 `set_size(220,72)` 归位一次（清洗历史版本插件可能存下的异常尺寸）。
3. **`show_main_window`（`lib.rs:1488`）**：托盘"显示/隐藏"重新显示时，若处于 docked 则恢复为 peek 几何（让用户看到完整胶囊），并 emit 现有 `OPEN_EVENTS` 不变。

#### 1.6 设置字段（Rust 侧）

`AppSettings` 增加两个字段（serde 默认值函数 + `normalize_settings` clamp，模式照抄现有字段）：

```rust
#[serde(default = "default_true")]
pub edge_dock_enabled: bool,          // 贴边收纳总开关
#[serde(default = "default_dock_hide_delay_ms")]
pub dock_hide_delay_ms: u32,          // 离开后收起延迟，clamp 200..=3000，默认 600
```

`save_settings` 已有的 `apply_runtime_settings` 中：若 `edge_dock_enabled` 变为 false 且当前 docked → 调 `undock_window` 逻辑。

### 阶段 2：前端 —— 交互状态机 + 灯珠渲染

#### 2.1 新模块 `src/dockModel.ts`（纯状态机，可单测）

```ts
export type DockEdge = "left" | "right" | "top";
export type DockUiMode = "none" | "hidden" | "peek";

export interface DockUiState {
  mode: DockUiMode;
  edge: DockEdge | null;
}

/** 事件驱动的纯转移函数：输入当前状态+事件，输出新状态+要执行的副作用指令 */
export type DockEvent =
  | { kind: "dockCheckResult"; edge: DockEdge | null }  // dock_check 返回
  | { kind: "pointerEnter" }                            // 悬停灯珠（已过 120ms 判定）
  | { kind: "pointerLeaveSettled" }                     // 离开且收起延迟已到
  | { kind: "dockDisabled" }                            // 设置关闭
  | { kind: "restored"; state: { edge: DockEdge } | null }; // 启动恢复

export type DockCommand =
  | { kind: "setMode"; mode: "hidden" | "peek" }  // → invoke set_dock_mode
  | { kind: "undock" }                            // → invoke undock_window
  | { kind: "closePanel" };                       // 收起时关闭事件面板

export function reduceDock(state: DockUiState, event: DockEvent): { state: DockUiState; commands: DockCommand[] }
```

为什么做成 reducer：悬停/离开/拖拽/设置变更的组合时序容易出竞态，纯函数转移 + 副作用指令列表可以用 node 测试穷举转移表，webview 里只剩"接事件、跑计时器、执行指令"的薄壳。

#### 2.2 `mainPanel.ts` 接线

新增一个 `setupDock(shell, deps)` 区块：

1. **拖拽结束检测**：`getCurrentWindow().onMoved(...)` 去抖 400ms → `invoke("dock_check", { panelExpandedHeight })` → 把结果喂给 reducer。
   - 自触发抑制：执行 `setMode`/`undock` 指令前置 `suppressMovedUntil = Date.now() + 500`，去抖回调里在抑制窗口内直接忽略（程序性移动也会触发 `onMoved`，不抑制会形成"吸附→Moved→再检测"的空转循环）。
2. **悬停/离开计时器**：
   - `document.documentElement` 上监听 `mouseenter` / `mouseleave`；
   - hidden 态 enter → 120ms 计时 → 发 `pointerEnter`；leave 取消计时；
   - peek 态 leave → `settings.dockHideDelayMs` 计时 → 发 `pointerLeaveSettled`；enter 取消计时；
   - 灯珠 `pointerdown` → 取消 peek 计时并 `startDragging()`（现有 pill 的 pointerdown 处理器从 `pill` 上移到 `shell` 上，灯珠和胶囊共用一份拖拽逻辑，`mainPanel.ts:300`）。
3. **指令执行**：`setMode` → `invoke("set_dock_mode")` 并更新 `shell.dataset.dockMode` / `dockEdge`；`closePanel` → 复用现有 `setPanelOpen(false)`（不再调 `resizePanelWindow`，几何由 `set_dock_mode` 一并处理）；`undock` → `invoke("undock_window")`。invoke 失败时回退 `mode: none` 渲染，保证 UI 不卡死在错误形态。
4. **启动恢复**：`loadSettings()` 后 `invoke("get_dock_state")` → 发 `restored` 事件（灯珠形态首帧对齐，避免闪一下完整胶囊）。
5. **设置联动**：现有 `SETTINGS_CHANGED` 监听里，若 `edgeDockEnabled === false` → 发 `dockDisabled`。
6. **面板联动**：docked-peek 下点击状态按钮展开事件面板 → `resizePanelWindow` 照常调 `set_panel_open`（后端已按 dock 分支贴边加高）。

## 5. 边界情况与风险

| 风险 | 处理 |
| --- | --- |
| 用户拖拽中途停顿 >400ms 被误判为松手，窗口提前吸附 | 吸附进 peek 而非 hidden，视觉变化小；继续拖走即恢复。P2 用 `GetAsyncKeyState` 消除误判 |
| 程序性 `set_position/set_size` 触发 `onMoved` 形成检测循环 | 前端 500ms 抑制窗口 + 后端幂等（目标几何一致则不动窗口）双保险 |
| window-state 插件恢复出 14×56 尺寸但状态丢失 | 插件限 `StateFlags::POSITION`；启动统一按 dock-state 归位（决策 6 + 1.5-2） |
| dock-state.json 损坏/显示器消失 | 解析失败→按未收纳处理；显示器找不到→回退主显示器同边缘并 clamp |
| 悬停展开与收起在边界抖动（resize 瞬间触发 mouseleave） | peek 展开方向朝屏幕内，光标必然仍在窗口内；leave 走 600ms 延迟计时，瞬时抖动会被 enter 取消 |
| 上边缘拖拽触发 Windows Aero Snap 最大化 | 窗口 `maximizable: false` + `resizable: false`，系统不会接管；验收项 10 实测确认 |
| 置顶被用户关闭后灯珠被其他窗口盖住 | 既有行为（胶囊同样会被盖），README 排障注明；不为此加特殊逻辑 |
| 事件面板展开时直接拖拽收纳 | `dock_check` 命中时强制 `closePanel`，收纳几何始终基于 72 高胶囊 |
| waiting 状态收纳后不够醒目 | 灯珠脉冲已覆盖；P2 可选"等待时自动展开"设置（默认关），本期不做 |
