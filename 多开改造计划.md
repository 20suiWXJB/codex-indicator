多实例监控执行方案

核心设计决策

实例身份怎么定? 一个"实例" = 一个活跃的 codex session jsonl 文件。理由:session 文件是 codex 自己保证一对一的事实来源,不需要猜 PID、不需要用户配置。

- 主键:session 文件名里的 uuid(rollout-*-<uuid>.jsonl)
- 显示名:session 文件首行 session_meta.payload.cwd 的最后一级目录名(即项目名)
- bridge 事件归属:payload 里有 session/thread id 就按 id 关联;只有 cwd 就归到该目录下最新活跃的实例(best-effort,同目录多开时可能归错,但不影响另一实例的独立显示)

"活跃"的定义:mtime 在 N 分钟内(默认 10 分钟,可配置)的 session 文件才算实例并显示;超时自动从列表消失。running 判定沿用现有 2 分钟 SESSION_RUNNING_TTL。

聚合规则(收起时那盏灯):waiting > error > interrupted > running > done > idle,副标题拼 2运行 1等待。

阶段 1:Rust 后端 — 多会话解析(核心,工作量最大)

改 src-tauri/src/lib.rs:

1. latest_codex_session_file(lib.rs:612)�� active_codex_session_files:不再 max_by_key 取一个,而是收集最近 3 个日目录下所有 mtime 在活跃 TTL 内的 jsonl,返回 Vec<(PathBuf, SystemTime, u64)>。
2. 单槽缓存 → 多槽:SESSION_PARSE_CACHE(lib.rs:519)从 Mutex<Option<SessionParseCache>> 改为 Mutex<HashMap<PathBuf, SessionParseCache>>,每个文件独立跑现有的增量解析状态机(增量逻辑 apply_appended_lines 原样复用,零改动);每轮清理已不活跃的缓存项,防止无限增长。
3. 解析 session_meta:全量重建缓存时顺带从首行取 cwd 和 session id,存进缓存项,作为实例 id 和 label。
4. 新数据结构与命令:
   struct InstanceStatus { id, label, cwd, status, summary, detail, updated_at }
   struct MultiStatusPayload { aggregate: StatusPayload, instances: Vec<InstanceStatus> }
   #[tauri::command] async fn get_statuses() -> MultiStatusPayload
4. 聚合按上面的优先级规则计算。get_status 保留并返回 aggregate(老测试和回退路径不破坏)。

验证:cargo test — 新增"两个活跃 session 文件 → 两个实例、聚合取高优先级"的用例;开两个真实 codex cli 目测。

阶段 2:bridge — 状态文件按实例拆分

改 hooks/agent-status-bridge.ps1:

1. 从 payload 提取 session-id/thread-id/cwd,算出实例键(id 优先,无 id 用 cwd 哈希,都没有用 "default")。
2. 状态写入 state/status/<实例键>.json(替代单一 status.json);events.jsonl 每行追加 instance 和 cwd 字段。
3. Rust 侧 read_effective_status_from_dirs(lib.rs:462)改为:读 state/status/*.json 得到一组 bridge 状态,按实例与阶段 1 的 session 推断合并——每个实例内部沿用现有优先级逻辑(bridge 的 waiting/error/interrupted/新鲜 done 压过 session 推断);过期的 bridge 状态文件(超过活跃 TTL)跳过并清理。

验证:npm run test:bridge — 新增"两个不同 cwd 的事件 → 两个状态文件互不覆盖"的用例。

阶段 3:前端 — 聚合灯 + 实例列表

1. statusPresenter.ts:加 InstanceStatus 类型;聚合副标题文案(2运行 1等待);实例行的状态文案复用现有 describeStatus 映射。
2. mainPanel.ts:refreshStatus 改调 get_statuses;收起态渲染聚合灯;展开面板顶部插入实例列表区(每行:状态点 + 项目名 + 状态词,hover 显示 cwd 全路径和 detail),下方保留最近事件,事件行前缀实例项目名。渲染指纹 lastRenderKey 机制扩展到实例列表。
3. style.css:实例行样式,与事件行同风格。

验证:npm run test:ui、npm run test:settings-page、npm run build。

阶段 4:收尾
