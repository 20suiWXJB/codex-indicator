## State Contract

`status.json` 使用最小稳定结构：

  ```json
  {
    "status": "waiting",
    "source": "codex",
    "event": "PermissionRequest",
    "summary": "Codex 正在等待批准",
    "detail": "powershell.exe ...",
    "updatedAt": "2026-07-02T16:50:00+08:00",
    "ttlMs": 0
  }
  ```

状态枚举固定为：

- `waiting`：黄色，Codex 权限审批中。
- `running`：蓝色，预留给后续 Claude/Codex running 事件。
- `done`：绿色，任务完成，前端 3.5 秒后本地显示为 `idle`。
- `error`：红色闪烁，hook 或 agent 失败。
- `interrupted`：橙红色，用户中断或 StopFailure 类事件。
- `idle`：灰色，无活动或状态文件不可读。

`events.jsonl` 每行一条事件：

  ```json
  {"status":"waiting","source":"codex","event":"PermissionRequest","summary":"Codex 正在等待批准","detail":"...","createdAt":"2026-07-02T16:50:00+08:00"}
  ```

损坏的 `status.json` 不抛到 UI；Rust command 返回 `idle` 加简短错误摘要，同时写日志。

## Implementation Plan

1. 基础清理与依赖收敛
    - 修改 `package.json`：移除 `vue`、`@vitejs/plugin-vue`、`vue-tsc`，保留 `vite`、`typescript`、`@tauri-apps/api`、`@tauri-apps/cli`。
    - `build` 脚本改为 `tsc --noEmit && vite build`。
    - 修改 `vite.config.ts`：移除 Vue plugin，保留 Tauri 推荐端口 `1420`。
    - 删除 Vue 模板文件和无关 svg 资源。

2. 前端悬浮窗
    - `index.html` 只保留一个 `#app` 容器。
    - `src/main.ts` 实现：
        - 每 500ms 调用 `get_status`。
        - 根据状态更新灯色、文本和 tooltip。
        - `done` 状态超过 3500ms 后仅前端降级显示 `idle`，不回写文件。
        - 点击/双击悬浮窗切换最近事件面板。
        - 调用 `get_recent_events` 显示最近 20 条。
    - `src/style.css` 实现：
        - 透明背景、紧凑悬浮灯。
        - 红灯闪烁、绿灯短暂停留、灰灯待机。
        - 事件面板不超过固定高度，可滚动。

3. Tauri 配置
    - 修改 `src-tauri/tauri.conf.json`：
        - 窗口宽高约 `220x72`，无边框、透明、置顶、不可最大化、默认右上角附近。
        - `decorations=false`、`transparent=true`、`alwaysOnTop=true`、`skipTaskbar=true`。

        - 打开日志目录
        - 退出
    - 托盘双击显示窗口并触发前端打开最近事件面板。
